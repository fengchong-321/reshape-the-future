# 第 25 周 - 需求分析 + 架构设计

## 学习目标
能够分析测试平台需求，设计合理的系统架构。

---

## 知识点清单

### 1. 竞品分析
**掌握程度**: 分析 5 个测试平台

**练习资源**:
- [TestRail](https://www.testrail.com/)
- [MeterSphere](https://metersphere.io/)
- [TestOps](https://smartbear.com/zipalign/)

**练习任务**:
- 注册试用 3 个平台
- 列出优缺点对比表
- 提取核心功能

---

### 2. 用户故事
**掌握程度**: 角色、场景、需求

**练习任务**:
- 定义用户角色
- 编写用户故事
- 排优先级

---

### 3. 功能规划
**掌握程度**: MVP、迭代计划

**练习任务**:
- 定义 MVP 功能
- 制定迭代计划

---

### 4. 技术选型
**掌握程度**: 前后端、数据库、消息队列

**练习任务**:
- 选择技术栈
- 说明选型理由

---

### 5. 架构设计
**掌握程度**: 分层、模块、接口

**练习任务**:
- 画系统架构图
- 定义模块边界

---

### 6. 数据库设计
**掌握程度**: ER 图、范式、索引

**练习任务**:
- 设计数据库 schema
- 画 ER 图

---

### 7. API 设计
**掌握程度**: RESTful 规范、版本

**练习任务**:
- 设计 API 接口
- 编写 API 文档

---

### 8. 文档撰写
**掌握程度**: 需求文档、设计文档

**练习任务**:
- 编写需求文档
- 编写设计文档

---

## 本周练习任务

### 必做任务

1. **竞品分析报告**
```markdown
# 测试平台竞品分析

## TestRail
- 优势：...
- 劣势：...
- 借鉴点：...

## MeterSphere
- 优势：...
- 劣势：...
- 借鉴点：...

## 总结
- 核心功能列表
- 差异化定位
```

2. **需求文档**
```markdown
# 测试平台需求文档

## 1. 项目概述
### 1.1 目标用户
### 1.2 核心价值
### 1.3 使用场景

## 2. 功能需求
### 2.1 用户管理
### 2.2 用例管理
### 2.3 测试执行
### 2.4 报告系统

## 3. 非功能需求
### 3.1 性能要求
### 3.2 安全要求
### 3.3 可用性

## 4. 约束条件
### 4.1 技术约束
### 4.2 时间约束
```

3. **架构设计文档**
```markdown
# 测试平台架构设计

## 1. 技术选型
- 前端：React + Ant Design
- 后端：FastAPI + SQLAlchemy
- 数据库：MySQL + Redis
- 消息队列：Celery + Redis

## 2. 系统架构
```
┌─────────────┐
│   Frontend  │
│   (React)   │

       │
┌──────▼──────┐
│     API     │
│  (FastAPI)  │
└──────┬──────┘
       │
┌──────▼──────┐
│   Database  │
│  (MySQL)    │
└─────────────┘
```

## 3. 数据库设计
### 3.1 用户表
### 3.2 用例表
### 3.3 测试计划表
### 3.4 测试报告表

## 4. API 设计
### 4.1 用户 API
### 4.2 用例 API
### 4.3 执行 API
```
