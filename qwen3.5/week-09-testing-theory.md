## 测试用例设计方法

### 等价类划分法
```
输入条件：年龄（1-120 岁）

有效等价类：
- 1-120 之间的整数

无效等价类：
- 小于 1 的数
- 大于 120 的数
- 非整数
- 空值
```

### 边界值分析法
```
输入条件：年龄（1-120 岁）

边界值：
- 最小值：1
- 略大于最小值：2
- 正常值：50
- 略小于最大值：119
- 最大值：120
```

### 因果图法
```
原因（输入条件）：
- C1: 用户名正确
- C2: 密码正确
- C3: 验证码正确

结果（输出）：
- E1: 登录成功
- E2: 登录失败

关系：
- E1 = C1 AND C2 AND C3
- E2 = NOT(C1 AND C2 AND C3)
```

## 面试考点

### 高频面试题
1. 测试流程是怎样的？
2. 如何设计测试用例？
3. 什么是等价类划分？
4. 什么是边界值分析？
5. 单元测试、集成测试、E2E 测试的区别？
6. 什么是冒烟测试、回归测试？
7. 如何描述一个 Bug？
8. 测试覆盖率如何计算？

### 场景题
```
题目：为微信发红包功能设计测试用例

思路：
1. 功能测试
   - 正常发红包
   - 金额边界（最小 0.01，最大 200）
   - 个数边界

2. 异常测试
   - 余额不足
   - 网络中断
   - 超时

3. 界面测试
   - UI 显示
   - 交互体验

4. 性能测试
   - 并发发红包
   - 响应时间

5. 安全测试
   - 金额篡改
   - 重放攻击
```
