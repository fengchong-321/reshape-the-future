# 第 33 周 - 性能测试理论

## 学习目标
掌握性能测试基础理论，能够制定性能测试计划。

---

## 知识点清单

### 1. 性能指标
**掌握程度**: QPS、RT、TP99、吞吐量

**练习任务**:
- 理解各指标含义
- 能计算指标

---

### 2. 测试类型
**掌握程度**: 负载、压力、稳定性、并发

**练习任务**:
- 区分各测试类型
- 理解适用场景

---

### 3. 测试流程
**掌握程度**: 计划→脚本→执行→分析

**练习任务**:
- 制定测试计划
- 执行测试

---

### 4. 基准测试
**掌握程度**: 基线、对比

**练习任务**:
- 建立性能基线
- 对比测试

---

### 5. 容量规划
**掌握程度**: 评估、预测

**练习任务**:
- 评估系统容量
- 预测瓶颈

---

### 6. 监控指标
**掌握程度**: CPU、内存、IO、网络

**练习任务**:
- 理解监控指标
- 能看懂监控

---

### 7. 瓶颈分析
**掌握程度**: 定位、根因

**练习任务**:
- 分析性能瓶颈
- 提出优化建议

---

### 8. 报告撰写
**掌握程度**: 结论、建议

**练习任务**:
- 写性能测试报告

---

## 本周练习任务

### 必做任务

1. **性能测试计划**
```markdown
# 性能测试计划

## 1. 测试目标
## 2. 测试范围
## 3. 测试环境
## 4. 测试工具
## 5. 测试场景
## 6. 性能指标
## 7. 时间安排
```

2. **建立性能基线**
```
# 对一个 API 进行基准测试
# 记录:
# - 响应时间
# - QPS
# - 错误率
# - 资源使用
```

3. **性能测试报告**
```markdown
# 性能测试报告

## 执行摘要
## 测试环境
## 测试结果
## 瓶颈分析
## 优化建议
```
