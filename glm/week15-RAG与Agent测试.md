## 二、知识点详解

### 2.1 进程与线程

```
进程（Process）：
- 程序的执行实例
- 资源分配的基本单位
- 拥有独立的地址空间
- 进程间通信需要特殊机制

线程（Thread）：
- 进程中的执行单元
- CPU 调度的基本单位
- 共享进程的地址空间
- 线程间通信更简单

进程状态：
- 新建（New）
- 就绪（Ready）
- 运行（Running）
- 阻塞（Blocked）
- 终止（Terminated）

状态转换：
新建 → 就绪 → 运行 → 阻塞 → 就绪 → 运行 → 终止
```

**面试常见问题：**

```
Q1: 进程和线程的区别？

A:
1. 本质：进程是资源分配单位，线程是调度单位
2. 地址空间：进程独立，线程共享
3. 通信：进程需要 IPC，线程可直接通信
4. 开销：进程创建/切换开销大，线程小
5. 健壮性：进程间隔离，一个崩溃不影响其他；线程一个崩溃可能导致进程崩溃

Q2: 进程间通信（IPC）方式？

A:
1. 管道（Pipe）：单向，父子进程
2. 命名管道（FIFO）：无亲缘关系进程
3. 消息队列（Message Queue）：内核管理的消息链表
4. 共享内存（Shared Memory）：最快，需要同步
5. 信号量（Semaphore）：同步互斥
6. 套接字（Socket）：跨网络通信

Q3: 线程同步方式？

A:
1. 互斥锁（Mutex）：独占访问
2. 读写锁（RW Lock）：读共享，写独占
3. 条件变量（Condition Variable）：等待条件
4. 信号量（Semaphore）：计数器
5. 自旋锁（Spin Lock）：忙等待

Q4: 什么是上下文切换？

A:
CPU 从一个进程/线程切换到另一个的过程。
需要保存当前上下文（寄存器、程序计数器等），恢复目标上下文。
开销：直接开销（保存/恢复）+ 间接开销（缓存失效）
```

### 2.2 内存管理

```
内存管理功能：
1. 内存分配与回收
2. 地址转换（逻辑地址 → 物理地址）
3. 内存保护
4. 内存扩充（虚拟内存）

虚拟内存：
- 使程序认为自己有连续的内存空间
- 实际可能分散在物理内存和磁盘
- 优点：
  * 进程可以大于物理内存
  * 进程隔离
  * 内存共享

分页（Paging）：
- 将内存分成固定大小的页（通常 4KB）
- 页表存储页号到页框号的映射
- 优点：无外部碎片
- 缺点：有内部碎片

分段（Segmentation）：
- 按逻辑单位划分（代码段、数据段、栈段）
- 段表存储段号到段起始地址的映射
- 优点：方便共享和保护
- 缺点：有外部碎片

段页式：
- 先分段，再分页
- 结合两者优点
```

**面试常见问题：**

```
Q1: 虚拟内存的作用？

A:
1. 扩展地址空间：进程可使用超过物理内存的空间
2. 进程隔离：每个进程有独立地址空间
3. 内存保护：设置访问权限
4. 内存共享：多个进程可共享同一物理页
5. 高效使用：只加载需要的页面

Q2: 页面置换算法？

A:
1. OPT（最优）：置换未来最久不用的（理论，无法实现）
2. FIFO（先进先出）：置换最早进入的
3. LRU（最近最少使用）：置换最久未访问的（常用）
4. LFU（最不常用）：置换访问次数最少的
5. Clock（时钟）：LRU 的近似实现

Q3: 什么是缺页中断？

A:
访问的页面不在物理内存中，触发缺页中断：
1. 挂起进程
2. 从磁盘加载页面到内存
3. 更新页表
4. 恢复进程执行

Q4: 内部碎片 vs 外部碎片？

A:
内部碎片：
- 分配的内存大于实际需要的
- 分页系统特有
- 无法利用

外部碎片：
- 内存中存在不连续的小块空闲区
- 分段系统特有
- 可以通过内存紧凑解决
```

### 2.3 进程调度

```
调度层次：
1. 长程调度（作业调度）：从外存调入内存
2. 中程调度（内存调度）：内存与磁盘交换
3. 短程调度（进程调度）：CPU 分配

调度算法：
1. FCFS（先来先服务）：简单，但可能导致"护航效应"
2. SJF（短作业优先）：平均等待时间最短
3. SRTF（最短剩余时间优先）：SJF 的抢占版本
4. RR（时间片轮转）：公平，适合分时系统
5. 优先级调度：可能导致饥饿
6. 多级反馈队列：综合多种算法

调度指标：
- 周转时间 = 完成时间 - 到达时间
- 响应时间 = 首次响应时间 - 到达时间
- 等待时间 = 周转时间 - 执行时间
- CPU 利用率 = 忙碌时间 / 总时间
- 吞吐量 = 完成进程数 / 单位时间
```

### 2.4 死锁

```
死锁四个必要条件：
1. 互斥：资源不能共享
2. 持有并等待：持有资源同时请求其他资源
3. 不可抢占：资源不能强制抢占
4. 循环等待：存在循环等待链

死锁处理策略：
1. 预防：破坏四个条件之一
   - 破坏互斥：不可能
   - 破坏持有并等待：一次性申请所有资源
   - 破坏不可抢占：允许抢占
   - 破坏循环等待：有序申请

2. 避免：银行家算法
   - 检查分配是否安全
   - 只在安全时分配

3. 检测与恢复：
   - 定期检测死锁
   - 发现后通过抢占或回滚恢复

4. 忽略：
   - 假设死锁很少发生
   - 大多数操作系统采用此策略
```

**面试常见问题：**

```
Q1: 死锁的必要条件？

A: 互斥、持有并等待、不可抢占、循环等待

Q2: 如何避免死锁？

A:
1. 银行家算法：分配前检查安全性
2. 资源有序分配：按顺序申请资源
3. 超时机制：等待超时后释放已持有资源
4. 一次性申请：同时申请所有需要的资源

Q3: 银行家算法原理？

A:
1. 维护 Available、Max、Allocation、Need 矩阵
2. 请求时检查是否小于 Need
3. 试探性分配
4. 检查是否存在安全序列
5. 安全则真正分配，否则回滚
```

### 2.5 Python 中的并发

```python
# ============================================
# 多线程
# ============================================
import threading

def worker(num):
    print(f"Worker {num}")

# 创建线程
threads = []
for i in range(5):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

# 线程锁
lock = threading.Lock()
counter = 0

def increment():
    global counter
    with lock:
        counter += 1

# ============================================
# 多进程
# ============================================
from multiprocessing import Process, Pool

def worker(num):
    return num * num

# 创建进程
processes = []
for i in range(5):
    p = Process(target=worker, args=(i,))
    processes.append(p)
    p.start()

for p in processes:
    p.join()

# 进程池
with Pool(4) as pool:
    results = pool.map(worker, range(10))

# ============================================
# 协程（asyncio）
# ============================================
import asyncio

async def fetch(url):
    print(f"Fetching {url}")
    await asyncio.sleep(1)
    return f"Result from {url}"

async def main():
    tasks = [
        fetch("url1"),
        fetch("url2"),
        fetch("url3")
    ]
    results = await asyncio.gather(*tasks)
    print(results)

asyncio.run(main())

# ============================================
# GIL（全局解释器锁）
# ============================================
"""
GIL 是什么？
- CPython 的互斥锁
- 同一时刻只有一个线程执行 Python 字节码
- 影响 CPU 密集型任务的并行性能

为什么需要 GIL？
- 简化 CPython 的内存管理
- 避免引用计数的竞争条件

如何绕过 GIL？
1. 使用多进程代替多线程
2. 使用 C 扩展
3. 使用其他 Python 实现（Jython、IronPython）
"""
```

### 题目1：进程和线程

```
Q: 进程和线程有什么区别？什么场景用多进程，什么场景用多线程？

A: 
多进程场景：
- CPU 密集型任务
- 需要高可靠性
- Python 中绕过 GIL

多线程场景：
- I/O 密集型任务
- 需要共享内存
- 资源受限环境
```

### 题目2：同步与互斥

```
Q: 什么是同步？什么是互斥？

A:
互斥：同一时刻只有一个进程/线程访问临界区
同步：多个进程/线程按特定顺序执行

互斥是同步的特殊情况。
```
